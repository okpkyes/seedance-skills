# 反瑕疵品质控制指南

Seedance 2.0 生成视频时常见瑕疵的分类、预防和修复方法。

---

## 使用时机

**用户遇到下列情况时，读取本文件：**
- ✅ "结果有瑕疵" / "不够精致" / "怎么修"
- ✅ 产品变形、扭曲
- ✅ 画面闪烁、不稳定
- ✅ 文字显示错误、鬼影
- ✅ 动作不自然、卡顿
- ✅ 角色一致性丢失

---

## 瑕疵分类与修复

### 问题 1：产品变形 / 形状不稳定

**症状**：
- 产品在不同帧中形状变化
- 液体/柔软物体变形严重
- 产品细节丢失

**原因**：
- Seedance 2.0 对物理模拟不够精确
- 提示词没有"锁定"产品形状

**预防提示词（必须加）**：
```
产品完整性锁：rigid form / no morphing / 形状不变 / 保持产品轮廓
产品细节：高清展示[具体细节，如"缝线/logo/纹理"]
物理稳定性：stable physics / 物理稳定
```

**修复方法**：
1. 使用**图生视频**（i2v）模式，上传产品高清图作为首帧
2. 提示词中明确"产品形状不变"
3. 避免复杂物理场景（如液体倾倒、布料飘动）

**平台选择**：
- ✅ 产品展示优先用**图生视频**（i2v），不要用文生视频（t2v）
- ✅ 强物理平台：Runway Gen-4.5 / Veo 3.1（但 Seedance 2.0 不支持）

---

### 问题 2：画面闪烁 / 不稳定

**症状**：
- 视频帧间闪烁
- 颜色跳动
- 画面抖动

**原因**：
- 视频生成模型的时间一致性不足
- 提示词过长或矛盾

**预防提示词**：
```
时间一致性：temporal consistency / 帧间稳定
禁止闪烁：no flickering / 稳定光照
平滑过渡：smooth transition
```

**修复方法**：
1. 缩短提示词，移除矛盾描述
2. 使用**视频延长**功能，而不是重新生成
3. 降低生成时长（5-8秒比10-15秒更稳定）

---

### 问题 3：文字显示错误 / 鬼影

**症状**：
- 文字拼写错误
- 文字重影、模糊
- 不需要的文字出现

**原因**：
- AI 视频模型对文字的理解不如图片模型
- 参考图中包含文字，被误识别

**预防提示词**：
```
文字要求：清晰显示"[准确文字内容]"，用双引号包住
禁止文字鬼影：no watermark / no ghost text / 禁止多余文字
文字位置：[具体位置，如"左上角/中央"]
```

**Seedance 2.0 文字显示技巧**：
- ✅ 用双引号包住要显示的文字：`"Seedream 5"`
- ✅ 文字要短（不超过5个中文字或10个英文字母）
- ✅ 避免复杂字体，用简单无衬线字体
- ❌ 不要试图显示长段落文字

**修复方法**：
- 如果文字错误，重新生成并简化文字内容
- 如果文字鬼影，检查参考图是否包含文字，移除或更换参考图

---

### 问题 4：动作不自然 / 卡顿

**症状**：
- 人物动作僵硬
- 运动不流畅，卡顿
- 多个动作之间跳跃

**原因**：
- 提示词描述了过多独立动作
- 运镜叠加太多

**预防提示词**：
```
单一主动作：focus on [一个核心动作]
流畅运动：smooth motion / 流畅过渡
避免动作堆叠：no multiple independent actions
```

**Seedance 2.0 运镜规则**：
- ✅ 一次最多 1-2 个运镜
- ❌ 不要同时写"zoom in while panning right and tilting up"（会烂掉）

**修复方法**：
1. 简化提示词，每次只描述 1 个核心动作
2. 使用**时间戳分镜法**，将长动作拆分为多个短段
3. 避免使用"快速切换""快切"等词汇，改为具体运镜描述

---

### 问题 5：角色一致性丢失

**症状**：
- 同一角色在不同帧中长相变化
- 参考图片中的角色特征丢失

**原因**：
- 提示词描述与参考图不一致
- 多个参考图风格冲突

**预防提示词**：
```
角色一致性：maintain character identity / 角色一致性
参考@图片1：@图片1的人物形象，保持脸部特征
禁止角色变形：no character morphing
```

**修复方法**：
1. 使用**首尾帧控制**，锁定角色外观
2. 提示词中明确引用参考图：`@图片1的人物形象`
3. 避免使用多个风格不同的参考图
4. 对于重要角色，使用**图生视频**（i2v）而不是文生视频

---

### 问题 6：音画不同步

**症状**：
- 台词与口型不对齐
- 音效与画面动作不同步

**原因**：
- Seedance 2.0 原生音效生成还不够精确
- 提示词中音效描述不具体

**预防提示词**：
```
音效同步：sound effect synchronized with [具体动作]
台词清晰：clear dialogue / 台词清晰
音效描述具体化：[动作]伴随[音效，如"脚步声/开门声"]
```

**修复方法**：
1. 音效描述要与画面动作一一对应
2. 台词要短（不超过15字），方便口型对齐
3. 如果音画不同步严重，后期用视频编辑软件调整

---

### 问题 7：生成超时 / 失败

**症状**：
- 视频生成超过 10 分钟还没完成
- 提示"生成失败"

**原因**：
- 提示词过长（>200字）
- 参考素材过多（超过 5 个文件）
- 平台服务器繁忙

**预防方法**：
1. **控制提示词长度**：15秒视频 ≤150字
2. **减少参考素材**：图片 ≤5张，视频 ≤2个
3. **避开高峰时段**：国内时间 20:00-22:00 是高峰

**修复方法**：
1. 缩短提示词，移除冗余描述
2. 减少参考素材数量
3. 换个时间再试

---

## 品质自检清单

**生成提示词前，检查下列项目：**

### 基础检查
- [ ] 提示词长度 ≤150字（15秒视频）
- [ ] 没有重复或矛盾的描述
- [ ] 运镜描述 ≤2个
- [ ] 参考素材数量 ≤5个

### 产品/角色一致性检查
- [ ] 产品展示优先用图生视频（i2v）
- [ ] 角色一致性用首尾帧控制
- [ ] 提示词中明确"形状不变""角色一致性"

### 技术检查
- [ ] 没有使用 Seedance 2.0 禁忌词汇（见 SKILL.md）
- [ ] 文字内容用双引号包住
- [ ] 音效描述与画面动作对应

### 禁止项检查
- [ ] 明确声明禁止水印、字幕、Logo
- [ ] 明确声明禁止产品变形（如适用）
- [ ] 明确声明禁止角色变形（如适用）

## 平台物理能力强弱速查

| 平台 | 强物理 | 弱物理 | 建议 |
|------|--------|--------|------|
| Runway Gen-4.5 | 液体、布料、烟雾 | 复杂机械 | 强物理场景首选 |
| Veo 3.1 | 真实物理、原生音讯 | 动画风格 | 写实场景 |
| Seedance 2.0 | 动画、特效 | 真实物理 | 动画/奇幻首选 |
| Kling 3.0 | 动作、舞蹈 | 细微表情 | 动作场景 |
| Vidu Q3 | 多角色 | 复杂物理 | 群戏 |

**原则**：
- 强物理场景（液体/布料/烟雾）→ 用 Runway / Veo
- 动画/特效场景 → 用 Seedance / Kling
- 不要在不合适的平台硬做复杂物理
